A new layer of the web

YourMine is not a platform. It is a distributed layer for applications and value. Instead of centralized services, YourMine enables a network where:

  • Applications are distributed, not hosted
  • Value is generated through participation
  • Identity is unique and non-transferable
  • Proof of participation defines access and rewards

There is no single system state. Only connected realities.

🔒
Safe by structure

A network of identity and participation

YourMine is built on a Soulnet — a network where identity and proof cannot be transferred or copied. This creates a system where:

  • Every user is uniquely bound to their actions
  • Participation has permanent attribution
  • Rewards cannot be detached from behavior
  • Sybil and duplication are structurally constrained

The network does not trust accounts. It trusts participation.

Apps that move through the network

Applications in YourMine are not installed from a store. They are distributed through the network itself.

  • Apps are modular and peer-to-peer
  • They propagate across nodes
  • Execution happens locally
  • Users decide what becomes active

The app layer is not centralized. It is emergent.

⬡
Spheres
Native JS apps living in the network. They run locally, share state via P2P, and expose a desktop icon and panel. Published via Build, scored by mining.
🎨
Themes
Full visual environments. Each theme owns its wallpaper collection, color system, and grid layout. Switchable via URL — /neural.theme.
⚡

Any web app — Bolt, Replit, Vercel, GitHub Pages — becomes a sphere instantly. Paste the URL. It runs in an isolated frame with full access to the YourMine bridge.
A universal app runtime

YourMine is a runtime layer for the open web. Any app hosted anywhere can connect to it and access identity, P2P networking, and persistent storage — without touching the YourMine codebase.

BRIDGE API — FROM YOUR APP
// Detect YourMine
const inYM = window.parent !== window;

// Receive profile on load
window.addEventListener('message', e => {
  if (e.data?.type === 'ym:ready')
    console.log(e.data.profile);

// Toast, storage, P2P
window.parent.postMessage({type:'ym:toast', msg:'Hello!'}, '*');
window.parent.postMessage({type:'ym:storage:set', key:'score', value:'42'}, '*');
window.parent.postMessage({type:'ym:p2p:broadcast', data:{x:1,y:2}}, '*');

The bridge is silent outside YourMine. Your app runs normally everywhere else. No SDK to install. No account to create. Paste a URL — it works.

Permission through score

In YourMine, access is not granted by an authority. It is earned through participation. The mining formula produces a score that determines permission levels across the network.

📊
Score = participation proof
Your score is derived from the amount burned, the patience rate chosen, and time elapsed. It is public, verifiable, and non-transferable.
🚪
Score gates publishing
Publishing a new sphere requires a score above a threshold. The ratio (score+1)/(laps+1) must exceed your last publication. This prevents spam and rewards consistency.
🔄
Upgrades always allowed
Updating an existing sphere only requires GitHub or wallet ownership. No score check. Improvement is always accessible.
PERMISSION CONDITION (new sphere)
(scorenow + 1) / (lapsnow + 1) > (scorelast + 1) / (lapslast + 1)
Ranking and discovery

Spheres and themes in YourMine are ordered by score descending. The highest-participation builders appear first. There is no editorial curation, no advertising, no sponsored placement.

  • Score is computed from the mining formula at time of publication
  • Higher burn + patience + elapsed time = higher score = higher rank
  • Score is frozen at merge time — it represents the effort at the moment of contribution
  • Updating a sphere resets its score to the current mining state
The ranking formula
rank = score ÷ laps — the ratio of claimable YM to time elapsed. A high-burn, patient miner who publishes at peak efficiency ranks highest. Gaming is structurally penalised.
